installation de rpmrebuild

Comme rpmrebuild travaille sur les packages rpm, il est fourni naturellement sous ce format.

quel fichier choisir ?

  • rpmrebuild-{version}.noarch.rpm : est un package prêt à installer, conçu pour les distributions qui utilisent rpm version 4.x : à utiliser de préférence
  • rpmrebuild-{version}.src.rpm : est un package source, pour ceux qui voudraient refabriquer le package rpm
  • rpmrebuild-{version}.tgz : les sources du logiciel
  • rpmrebuild-{version}.tgz.sig : la signature des sources

vérifier le package

Tous les packages rpm sont signés avec ma clef gpg. Je vous recommande fortement de vérifier tout fichier téléchargé avec :
  • pour les packages rpm, la commande rpm -K rpmrebuild*.rpm
  • pour les sources, la commande gpg --verify rpmrebuild-{version}.tgz.sig rpmrebuild-{version}.tgz

installation

Rpmrebuild est codé en shell (bash), donc il ne dépends pas de librairies, ni de l'architecture.

avec rpm version 4

Il s'installe avec la commande : rpm -Uvh rpmrebuild-{version}.noarch.rpm.

avec rpm version 3

Nous n'avons pas de machine utilisant cette version de rpm. Il vous faudra donc fabriquer votre propre package rpm à partir du fichier source en utilisant la commande : rpm -tb rpmrebuild-{version}.tgz, puis l'installer avec une commande rpm -Uvh rpmrebuild-{version}.noarch.rpm.

en cas de problème

vous pouvez fabriquer un package rpm pour votre distribution avec l'une des méthodes suivantes :
  • rpmbuild --rebuild rpmrebuild-{version}.src.rpm (rpm version 4)
  • rpmbuild -tb rpmrebuild-{version}.tgz (rpm version 4)
  • rpm -tb rpmrebuild-{version}.tgz (rpm version 3
  • si ça ne marche toujours pas, envoyez-nous un rapport de bug
